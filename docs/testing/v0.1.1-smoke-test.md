# v0.1.1 Smoke Test Checklist

## Goal

Validate the release build in an isolated app-data directory without touching the main local database in `%LocalAppData%\SubscriptionTracker`.

## Isolated App Data

The application supports an override directory for release validation:

- environment variable: `SUBSCRIPTION_TRACKER_APPDATA_DIR`

When this variable is set, the application stores:

- `settings.json`
- `subscription_tracker.db`

inside the provided directory instead of `%LocalAppData%\SubscriptionTracker`.

## First-Run Validation

1. Prepare an empty folder, for example:
   - `C:\temp\subscription-tracker-smoke`
2. Start the release build with:

```powershell
$env:SUBSCRIPTION_TRACKER_APPDATA_DIR = "C:\temp\subscription-tracker-smoke"
.\artifacts\publish\wpf\SubscriptionTracker.Wpf.exe
```

3. Confirm expected first-run behavior:
   - the application starts successfully
   - `subscription_tracker.db` is created
   - default categories are seeded
   - sample subscriptions are visible on the dashboard
   - no existing production database is touched

## Manual Smoke Checklist

- Launch the application from the release publish folder.
- Open `Dashboard` and confirm KPIs and charts render.
- Open `Subscriptions` and create a new subscription.
- Mark a payment as paid.
- Skip the next payment for another subscription.
- Disable and re-enable a subscription.
- Open `Calendar` and verify filters and monthly groups work.
- Open `History` and verify filters, search, and status rendering work.
- Open `Analytics` and verify all charts render without errors.
- Open `Settings` and change:
  - language
  - theme
  - base currency
  - reminder interval
- Save settings and restart the application.
- Export to Excel and confirm the file is created.
- Create a backup and restore it from the backup file.
- Confirm reminder startup behavior does not close the app.
- Confirm custom dialogs and toast fallback work as expected.

## Expected Artifacts

- `subscription_tracker.db` exists after first run
- `settings.json` appears after saving settings
- no crash during startup, chart rendering, export, backup, or restore
